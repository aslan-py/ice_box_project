# Ice Booking API ⛸️
Ice Booking API — это высокопроизводительный асинхронный сервис на базе FastAPI, предназначенный для автоматизации процессов ледового катка. Система управляет динамическим расписанием (слотами), бронированиями в реальном времени и пользователями.  
Управление расписанием (IceBox):
- Публичный доступ: Просмотр доступных сеансов и статуса заполненности.
- Администрирование: Динамическое создание и удаление временных слотов.
- Автоматизация: Контроль вместимости (capacity) и автоматическое переключение флага is_full.
   
Система бронирования:
- Smart Booking: Регистрация заявок с автоматической проверкой доступных мест.
- Личный кабинет: Просмотр истории и активных бронирований.
- Отмена: Безопасное удаление брони с мгновенным освобождением места в слоте.
  
Регистрация и аутентификация пользователей:
- Auth: Асинхронная аутентификация через JWT (JSON Web Tokens).
- Безопасность: Регистрация, логирование, управление сессиями (Logout).
- Profile: Полноценный CRUD для профиля пользователя (просмотр и обновление данных).
## 🌟 Основной функционал
Регистрация и аутентификация (Auth)  
- Двухфакторная верификация: Пользователь вводит телефон и email, получая короткий код подтверждения (отправляется по SMS и на email для надежности).
- JWT Авторизация: При вводе телефона и кода выдается JWT-токен (PyJWT).
- Безопасность: Встроены проверки существования телефона/почты и защита от подмены данных.
- Ролевая модель: Все новые пользователи получают роль 'USER' и стандартное имя 'Новичек'.
- `Тестовый эндпоинт:` Специальный эндпоинт для передачи прав администратора пользователям с ролью 'USER' (только для тестирования API, на продакшене скрывается). 

Управление пользователями (Profile)  
- Пользователи: Могут просматривать свои данные, деактивировать аккаунт и редактировать ограниченный набор полей (телефон, почта, имя). Остальные поля защищены от изменений.
- Администраторы: Имеют полный доступ ко всем пользователям и могут редактировать любые поля в их карточках.  

Управление расписанием (IceBox)
- Публичный доступ: Доступно всем (даже без регистрации). Отображаются только активные слоты в настоящем времени. Прошлые и неактивные слоты скрыты.  
- Администрирование: Создание, изменение параметров и просмотр всех слотов с фильтрацией доступны только администраторам.
- Уведомления: При деактивации забронированного слота администратором, пользователи автоматически получают уведомления об отмене по почте и SMS. В проекте используется сервис смс рассылок https://smsaero.ru/

Система бронирования (Smart Booking)  
- Бронирование: Зарегистрированные пользователи могут бронировать доступные слоты и безопасно отменять (удалять) свою бронь.
- Просмотр: Пользователи видят только свою историю и активные бронирования. Администраторы имеют доступ ко всем бронированиям в системе.

## 🛠 Технологии
* **Python 3.12**
* **FastAPI** — REST API
* **SQLAlchemy (async)** — работа с базой данных
* **Alembic** — миграции БД
* **Pydantic** — валидация данных
* **SQLite** — легковесная асинхронная БД для хранения данных
* **Redis** — кэширование данных
* **Celery** — фреймворк для выполнения фоновых и отложенных задач (рассылка SMS/Email)
* **Rabbit MQ** — брокер сообщений для Celery 
* **Smsaero.ru** — сервис отправщик смс через API ключ

## 🔗 Примеры API-запросов
**Аутентификация: POST/auth/request-code**   
Запрос:
```json
{
"email": "user@example.com",
"phone": "79998887766"
}
```
Ответ:  
```json
{
    "status": "succes",
    "message": "Код отправили на почту и по смс(ожидание смс до 5 минут)"
}
```

**изменить даныне пользователя админом: PATCH /user/{user_id=1}**      
Запрос:
```json

{
  "is_active": true
}
```
Ответ:  
```json
{
  "name": "Aslan",
  "phone": "79998887766",
  "email": "user@example.com",
  "id": 1,
  "role": "ADMIN",
  "is_active": true
}
```
